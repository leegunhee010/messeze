# messeze — 집/다른 PC에서 이어서 작업하기 (최신)

이 폴더 하나면 어디서든 편집·빌드·배포를 이어갈 수 있어요.

## 1. 라이브 / 저장소
- **라이브 사이트**: https://mina0313.github.io/messeze/
- **GitHub**: https://github.com/mina0313/messeze (본인 계정 `mina0313`, 브랜치 `master`, GitHub Pages)
- 원본 인수인계 저장소 `leegunhee010/messeze` 는 **다른 사람 것 → push 금지**

## 2. 집 PC에서 시작하는 가장 쉬운 방법 (clone 방식 권장)
```bash
git clone https://github.com/mina0313/messeze.git
cd messeze
# 편집 → 빌드 → 아래로 배포
git add -A
git commit -m "수정 내용"
git push origin master        # 1~2분 뒤 라이브 반영
```
- 최초 push 때 GitHub 로그인(mina0313) 1회 → 이후 저장됨
- **커밋 메시지에 큰따옴표(")·화살표(->) 쓰지 말 것** (PowerShell에서 파싱 깨짐). `->` 대신 `에서 ~로` 처럼 한글로.

> ⚠️ 지금까지는 편집본(`messeze-인수인계`)에서 고쳐 배포본(`messeze-live`)으로 복사해 push하는 2폴더 방식을 썼는데, 집에서는 **clone 한 폴더 하나에서 바로 편집·push** 하는 게 훨씬 간단해요.

## 3. 폴더 구조 / 편집 규칙
- **직접 편집**: `index.html`, `services.html`, `pricing.html`, `check.html`, `faq.html`
- **생성물(직접 수정 금지)**: `services/`, `blog/`, `glossary/` — 각각 아래 py로 생성. 내용 수정은 py 안 배열에서 하고 **재빌드**.
  - `build-services.py` → services/*.html (서비스 상세 6개). 데이터는 `S`(s(...)) 배열, 히어로 비주얼·모션도 여기.
  - `build-blog.py` → blog/ (글 15편). 데이터는 `POSTS` 배열, 커버는 `_PHOTOS`/`_SVG_COVERS`+`COVERS`, 본문 도식은 `FIGS`.
  - `build-glossary.py` → glossary/ (용어 74개). `T` 배열.
- **메가메뉴는 8곳 중복** — 항목 바꾸면 `index/services/pricing/check/faq.html` + `build-services.py`+`build-blog.py`+`build-glossary.py` 전부 동기화 후 재빌드. 상단 네비도 같은 곳들 동기화.

## 4. 도구 / 빌드 / 미리보기
- **Node·git** 필요. **Python**: 집 PC에 있으면 `python build-*.py`. 없으면 python.org에서 "Windows embeddable package" 받아 압축 풀고 그 python.exe로 실행. (회사 PC에선 `C:\Users\mycom\Desktop\pyembed\python.exe` 사용했음)
- **빌드**: `python build-services.py` / `python build-blog.py` / `python build-glossary.py`
- **로컬 미리보기**: 폴더에서 `python -m http.server 5678` → 브라우저 `http://localhost:5678/index.html`

## 5. 지금까지 완성된 것 (2026-07 작업)
**메인(index.html)**
- 히어로 좌우 2단(텍스트←/대시보드카드→ 슬라이드) + 카드 floatY
- 스크롤 리빌은 **토글식**(위·아래 오갈 때마다 재생). 히어로 하단 AI플랫폼 칩 4+2 2줄.
- "왜 지금인가" 1·2·3 핀 하이라이트(3번 마크: 비활성 회색/활성 파랑 스윕)
- "노출 비교" 미노출 행 강조 + 깜빡임(링 blink + 미노출 배지 펄스)
- **Market Opportunity 차트**(GEO vs 전통검색, 2027 골든크로스) + 하단 통계카드 3종(-25%/3.2x/90%)
- **HOW 4단계 프로세스 섹션(#how)** — 진단/정비/축적/리포트, 스크롤 시 하나씩 등장 (searchpolaris 참고)
- 스탯밴드(3개월의 3 흰색), 요금 카드(가격 카운트업), FAQ(항목별 리빌·단일 오픈 아코디언), 최종 CTA

**요금(pricing.html)**
- 히어로 부제 2줄, 요금 카드 호버 리프트 + 스크롤 순차 등장
- "모든 플랜 공통" 4카드 스크롤 순차 강조
- **비교표 핀 스포트라이트** — 섹션 고정 후 스크롤에 따라 강조 열이 소상공인→기업형→엔터로 이동(기업형만 활성 시 진한 잉크)

**서비스(services.html + services/*)**
- 서비스 목록 페이지 스크롤 모션, "이어집니다" 관련 카드 호버 강화
- 서비스 상세 히어로: 좌우 슬라이드 + 카드 floatY + **내부 그래프 모션**(게이지·막대·노드·달력·클립 순차)
- fit 섹션 제목 통일: "이런 상황이라면, 시작할 때입니다"
- 비교표(왜 메세지인가) messeze 열 전체 색상(다크 헤더+연블루 셀)
- 자사블로그 비주얼: 발행을 **달력**으로, 언론: 매체 클립 3개

**블로그(build-blog.py, 15편)**
- 신규 8편: 색인vs인용(구글·ChatGPT 로고 커버)/GEO 측정/llms.txt/AI 허위정보/해외바이어 AI/수출바우처/리뉴얼 사례/보도자료 사례
- 모든 글 본문 도식(FIGS) + 커버(사진 또는 브랜드 SVG). 카테고리: AI가시성·언론홍보·수출해외PR·사례

**FAQ 페이지(faq.html)**
- 좌측 카테고리 사이드바 + 스크롤스파이, 공통 3 + 서비스별 6 + 도입·운영·성과(weekerp 참고) = **10카테고리 43문항**, 단일 오픈 아코디언

**용어사전**: 74개 / **check.html**: 점수 구조 막대 스크롤 채우기 모션 / **메가메뉴**: 서비스 컬럼 맨 위 "서비스 전체 보기" 링크

## 6. 알아두면 좋은 점 / 주의
- **모션이 안 보이면**: 이 PC 특성상 Windows 애니메이션 효과가 꺼져 있으면 CSS 모션이 전부 안 나옴 → 설정 > 접근성 > 시각 효과 > 애니메이션 효과 ON. (집 PC도 확인)
- 서비스 히어로 내부 모션은 **페이지 로드 시 재생**(위쪽이라 놓치기 쉬움). 스크롤 살짝 올렸다 내리면 다시 재생.
- 블로그 사진 커버 일부는 Unsplash 핫링크라 **내용 적합성 미확인**(요가 사진 사건 있었음). 어색하면 `_PHOTOS`에서 교체하거나 `_SVG_COVERS`로 SVG 커버 사용.
